## 1. 当前项目总体结构

当前项目已经从“`K230 视觉 + UART 发给下位机`”改成了“`K230 视觉 + 本机 PID 步进控制`”，并额外补了一套“双核异构迁移骨架”。

当前有两种运行方式：

1. 单进程本机控制
   - 视觉脚本直接调用本机 PID + PWM 步进控制
   - 由 `control_backends.py` 中的 `LocalMotorControlBackend` 执行

2. 双核迁移预备模式
   - 小核视觉侧只输出控制消息
   - 大核实时侧负责 PID、PWM、使能与保护
   - 当前仓库里只提供了协议和骨架，还没有真正接入 K230 SDK 工程

## 2. 主要文件关系

### 2.1 启动入口

#### `main.py`

作用：

- 项目启动入口
- 根据 `FLAG` 选择运行模式
- 支持通过 `GPIO20` 启动时单键轮换模式

关系：

- `FLAG=2/3` -> `stand_aiming.py`
- `FLAG=4/5` -> `move_aiming.py`
- `FLAG=6` -> `circle_mode.py`
- `FLAG=0` -> `calibrate.py`

注意：

- 当前 `main.py` 已经不再内嵌模式源码，而是直接导入外部文件
- 后续改 `stand_aiming.py / move_aiming.py / circle_mode.py` 会直接生效

---

### 2.2 三个主运行模式

#### `stand_aiming.py`

作用：

- 识别矩形框中心
- 计算目标点与矩形中心的像素误差
- 输出给控制后端

控制量单位：

- `pixel`

典型用途：

- 静态对准

重点关注：

- `STEPPER_AXIS_OVERRIDES`
- `ALIGNED_TOLERANCE_PX`
- `TARGET_POINT_OFFSET_Z`

---

#### `move_aiming.py`

作用：

- 识别靶面、靶心、激光点
- 计算激光点相对靶心的物理误差
- 可输出普通瞄准误差或圆轨迹误差

控制量单位：

- `cm`

典型用途：

- 常规跟踪
- 移动目标校正

重点关注：

- `STEPPER_AXIS_OVERRIDES`
- `CIRCLE_RADIUS_CM`
- `RED_THRESHOLD / BLACK_THRESHOLD / VIOLET_THRESHOLD`

---

#### `circle_mode.py`

作用：

- 圆环模式专用
- 生成圆周目标点
- 计算激光点与目标点之间误差
- 带状态机：`IDLE -> WAITING -> RUNNING`

控制量单位：

- `cm`

重点关注：

- `STEPPER_AXIS_OVERRIDES`
- `START_ALIGN_TOL_CM`
- `START_ALIGN_HOLD_FRAMES`
- `ERROR_JUMP_MAX_CM`
- `ERROR_JUMP_REJECT_FRAMES`

---

### 2.3 公共控制层

#### `k230_common.py`

作用：

- 标定参数读写
- 双轴步进控制公共层
- 单轴 PID 控制与 PWM 输出

核心类：

- `StepperAxis`
- `DualAxisStepperController`

控制链路：

1. 模式脚本计算误差
2. 后端把误差传入控制器
3. `StepperAxis` 计算 PID 输出
4. PID 输出变成目标步频
5. PWM 输出步进脉冲
6. `DIR / EN / STEP` 控制驱动器

这里是当前实时控制的核心文件。

---

### 2.4 控制后端抽象层

#### `control_backends.py`

作用：

- 统一“本机控制”和“IPC 发包”两种控制方式

当前后端：

- `LocalMotorControlBackend`
- `SharefsIPCControlBackend`

意义：

- 模式脚本不需要知道控制是在本机执行，还是发给大核执行
- 后续切到双核时，不需要重写三种视觉脚本

---

### 2.5 双核通信协议

#### `dual_core_ipc.py`

作用：

- 定义小核视觉侧发给大核实时侧的统一控制消息格式
- 当前使用 ShareFS JSON 文件作为占位传输通道

默认输出路径：

```text
/sharefs/k230_dual_core/vision_control_latest.json
```

消息内容包括：

- `mode`
- `unit`
- `valid`
- `control_enabled`
- `sync_ok`
- `aligned`
- `state`
- `error_x_milli`
- `error_y_milli`
- `target_x_milli`
- `target_y_milli`

---

### 2.6 双核配置入口

#### `dual_core_config.py`

作用：

- 统一切换当前控制后端

当前可选值：

```python
CONTROL_BACKEND = "local"
CONTROL_BACKEND = "sharefs_ipc"
```

说明：

- `local`：当前最稳定，视觉与控制仍在同一运行环境
- `sharefs_ipc`：小核只发控制包，为双核迁移做准备

这是当前最重要的运行模式切换入口之一。

---

### 2.7 双核迁移骨架

#### `dual_core/`

作用：

- 放置双核异构迁移相关骨架文件

目录关系：

- `dual_core/README.md`
  - 双核骨架总览
- `dual_core/little_core/README.md`
  - 小核侧说明
- `dual_core/big_core/include/vision_control_protocol.h`
  - 大核侧控制协议结构体
- `dual_core/big_core/include/vision_pid_controller.h`
  - 大核侧 PID 控制器接口
- `dual_core/big_core/src/vision_pid_controller.c`
  - 大核侧 PID 控制骨架实现
- `dual_core/big_core/src/sharefs_json_receiver_stub.c`
  - ShareFS 接收占位实现

注意：

- 这里还不是完整可编译的 K230 SDK 工程
- 它只是迁移时的接口骨架

## 4. 当前真正需要调试的部分

下面这些是上板后最常需要改的地方。

### 4.1 步进电机引脚定义

文件：

- `k230_common.py`

位置：

- `DEFAULT_STEPPER_AXES`

需要确认：

- `step_board_pin`
- `dir_board_pin`
- `enable_board_pin`
- `dir_gpio_num`
- `enable_gpio_num`
- `enable_active_low`
- `dir_invert`

说明：

- 如果方向反了，优先改 `dir_invert`
- 如果模式层定义了 `command_sign`，也可能通过改 `command_sign` 修正方向

---

### 4.2 PID 参数

文件：

- `stand_aiming.py`
- `move_aiming.py`
- `circle_mode.py`

位置：

- 各文件顶部的 `STEPPER_AXIS_OVERRIDES`

当前每轴可调参数：

- `pid_kp`
- `pid_ki`
- `pid_kd`
- `integral_limit`
- `integral_active_error`
- `deadband`
- `error_full_scale`
- `min_freq`
- `max_freq`
- `ramp_hz_per_s`

建议调试顺序：

1. 先只调 `kp`
2. 再少量加入 `kd`
3. 最后再加 `ki`
4. 最后调整 `deadband` 与 `min_freq`

风险：

- `kp` 过大：来回振荡
- `ki` 过大：缓慢积累后突然冲过头
- `kd` 过大：噪声放大，电机抖动

---

### 4.3 视觉阈值

文件：

- `move_aiming.py`
- `circle_mode.py`
- `calibrate.py`

需要调的典型阈值：

- `RED_THRESHOLD`
- `BLACK_THRESHOLD`
- `VIOLET_THRESHOLD`
- `RING_THRESHOLD`

说明：

- 如果靶心识别错、激光点丢失、圆环检测飘，优先检查阈值
- 若现场光照变化大，应优先用 `calibrate.py` 重新标定

---

### 4.4 对准判定与状态机

文件：

- `stand_aiming.py`
- `circle_mode.py`

典型参数：

- `ALIGNED_TOLERANCE_PX`
- `START_ALIGN_TOL_CM`
- `START_ALIGN_HOLD_FRAMES`

说明：

- 太严格：系统长时间不进入运行态
- 太宽松：误差还很大就开始跑

---

### 4.5 控制后端切换

文件：

- `dual_core_config.py`

说明：

- 想继续单机调 PID：保持 `local`
- 想开始做双核联调：改成 `sharefs_ipc`

注意：

- 改成 `sharefs_ipc` 后，视觉侧不再真正驱动电机，而是输出控制包
- 这时如果大核侧还没有接收程序，电机不会动作

## 5. 当前文件之间的调用关系

简化调用链如下：

```text
main.py
  -> stand_aiming.py / move_aiming.py / circle_mode.py
      -> control_backends.py
          -> local:
               k230_common.py
          -> sharefs_ipc:
               dual_core_ipc.py
```

未来双核目标链路如下：

```text
little core:
  stand_aiming.py / move_aiming.py / circle_mode.py
    -> control_backends.py
      -> dual_core_ipc.py
        -> ShareFS / IPC

big core:
  IPC receiver
    -> vision_control_protocol.h
    -> vision_pid_controller.c
    -> PWM / DIR / EN
```

## 6. 调试时的推荐顺序

建议按下面顺序调试，不要一开始就同时改视觉、PID、双核。

### 第一步：本机单进程跑通

要求：

- `dual_core_config.py` 使用 `local`
- 电机能转
- 方向正确
- 误差减小时，电机趋于停下

优先检查：

- `k230_common.py` 引脚定义
- `STEPPER_AXIS_OVERRIDES`

---

### 第二步：调好视觉识别

要求：

- 靶面、靶心、激光点识别稳定
- 状态机不乱跳
- 误差数值方向正确

优先检查：

- 颜色阈值
- ROI
- 圆环检测参数

---

### 第三步：调 PID

要求：

- 不明显振荡
- 有足够跟随速度
- 丢目标时能快速停机

优先检查：

- `pid_kp / pid_ki / pid_kd`
- `deadband`
- `min_freq`
- `max_freq`
- `ramp_hz_per_s`

---

### 第四步：切换到双核预备模式

要求：

- `dual_core_config.py` 改成 `sharefs_ipc`
- 小核能持续输出控制包
- 大核接收侧能读到包

优先检查：

- ShareFS 路径是否可用
- JSON 文件是否生成
- 序号 `seq` 是否递增

---

### 第五步：迁移到真正大核 RT-Smart 工程

要求：

- 不再依赖 JSON 文件轮询
- 替换成正式核间通信
- 大核固定周期执行 PID

这一步已经超出当前纯 Python 仓库范围。

## 7. 目前最容易踩坑的地方

### 7.1 以为当前仓库已经完成真正双核

不是。

当前只是：

- 小核侧消息接口已整理
- 大核侧控制骨架已整理

但还没有：

- 真正的 K230 SDK 大核工程
- 正式核间通信接入
- 大核实机 PWM 控制闭环任务

---

### 7.2 `sharefs_ipc` 打开后电机不转

这是正常现象，前提是你还没有接好大核接收程序。

因为这时视觉侧只发消息，不直接控电机。

---

### 7.3 PID 调好了但识别误差方向错

这会表现成：

- 越调越偏
- 电机始终朝反方向修正

优先排查：

- `command_sign`
- `dir_invert`
- 图像坐标系与云台方向映射关系

---

### 7.4 视觉噪声导致 PID 抖动

原因常见有：

- 激光点检测不稳
- 靶心识别跳动
- 光照变化大

处理建议：

- 先稳视觉，再调 PID
- 必要时增大 `deadband`
- 适当减小 `kd`

---

### 7.5 步进最低频率太高

会导致：

- 小误差时也抖
- 接近目标点时来回抽动

优先调：

- `min_freq`
- `deadband`
