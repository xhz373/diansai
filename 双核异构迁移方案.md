# K230 双核异构迁移方案

本文对应当前仓库代码状态，更新时间为 `2026-07-17`。

## 1. 当前已完成

当前 Python 工程已经把实时控制逻辑集中到 `k230_common.py`：

- `StepperAxis`：单轴步进电机控制，包含 `PID + 限频 + 斜坡`
- `DualAxisStepperController`：双轴控制聚合
- `stand_aiming.py / move_aiming.py / circle_mode.py`：只负责视觉检测与误差生成

这意味着：

- 实时控制边界已经从模式脚本中剥离出来
- 后续可以把 `k230_common.py` 中的控制层迁移到 RT-Smart 大核侧
- 小核侧只保留图像、识别、状态机和误差发送

## 2. 目标异构分工

建议按以下方式分工：

### 大核

执行高实时任务：

- 云台双轴 PID
- 步进方向控制
- PWM 步频输出
- 使能/停机保护
- 对上一次目标误差进行高频插值或保持

### 小核

执行非实时任务：

- 摄像头采集
- 目标检测
- 光斑检测
- 靶心定位
- 模式切换与状态管理
- 把误差 `error_x / error_y` 发送给大核

## 3. 推荐数据接口

建议小核只给大核发送简化后的控制目标：

```text
timestamp_ms,error_x,error_y,mode,valid
```

字段建议：

- `timestamp_ms`：视觉帧时间戳
- `error_x`：X 轴控制误差
- `error_y`：Y 轴控制误差
- `mode`：`stand` / `aim` / `circle`
- `valid`：目标是否可信，失效时大核立即停机

不建议把完整视觉逻辑搬到大核。

## 4. 迁移步骤

1. 把 `k230_common.py` 中 PID 与步进输出逻辑翻译为 K230 SDK C/C++ 版本。
2. 在大核 RT-Smart 侧建立高频控制循环，固定周期执行 PID。
3. 在小核 Linux/CanMV 侧保留当前 Python 视觉代码，仅输出误差。
4. 用核间通信把误差从小核发送到大核。
5. 大核在视觉失联、超时或 `valid=0` 时立即停脉冲并按需断使能。

## 5. 当前限制

当前仓库只有 CanMV Python 脚本，没有：

- RT-Smart 大核工程
- 小核与大核的 IPC 工程
- K230 SDK 构建脚本

因此当前仓库内可以完成：

- PID 控制逻辑收敛
- 视觉层与控制层职责拆分

但不能仅靠这些 Python 文件，直接实现真正的大小核分别运行。
